## **📄 phase3.md: Scoring Engine & Fail-safe Notification**

### **1\. 목표 및 설계 원칙**

* **Scoring Engine:** 사용자 설정 가중치(Map)를 기반으로 시장 전체 순위 자동 산출.  
* **Robustness:** 불완전한 시장 데이터(null, 0, 음수)에 대한 철저한 예외 처리.

### **2\. 세부 수행 작업 (Detailed Tasks)**

1. **스코어링 로직 구현:** 지표별(PER, ROE 등) 값을 0\~100점 사이로 정규화(Min-Max Scaling)하고 사용자가 부여한 가중치(%)를 곱해 합산 점수 도출.  
2. **데이터 예외 처리(중요):**  
   * PER이 0 이하이거나 ROE가 null인 경우: 해당 종목은 스코어링에서 제외하지 않고 **'N/A' 상태로 처리하거나 최하점(0점)을 부여**하여 앱 크래시 방지.  
   * 데이터 누락 시 기본값(Default Value) 설정 로직 포함.  
3. **민감도(Sensitivity) 시스템:** 알림 발생 임계값을 High(상위 10% 이탈), Medium(20%), Low(30%)로 구분하여 유저가 선택 가능하도록 설계.  
4. **로컬 알림 서비스:** flutter\_local\_notifications를 초기화하고 전략 이탈 시 "보유 종목 \[삼성전자\]가 전략 순위권에서 이탈했습니다" 메시지 발송 함수 작성.  
5. **백그라운드 모니터링:** WorkManager를 사용하여 장 마감 시간(예: 오후 4시)에 하루 1회 백그라운드에서 스코어링을 재계산하고 알림을 트리거하는 작업 등록.

### **3\. 검증 및 디버깅 (Verification & Debugging)**

* **비정상 데이터 주입:** 특정 종목의 PER을 0으로, ROE를 null로 강제 설정한 뒤에도 스코어링 엔진이 중단 없이 돌아가는지 테스트.  
* **알림 시뮬레이션:** 백그라운드 태스크를 강제 실행(WorkManager.initialize)하여 실제 푸시 알림이 화면 상단에 뜨는지 확인.  
* **민감도 로직 검증:** 민감도를 'High'로 설정했을 때와 'Low'로 설정했을 때 동일한 순위 하락 상황에서 알림 발생 여부가 달라지는지 확인.

## **📄 phase4.md: High-Fidelity UI & Normalization Chart**

### **1\. 목표 및 설계 원칙**

* **Data Visualization:** 단위가 다른 지표들을 시각적으로 왜곡 없이 비교.  
* **Smart Tagging:** 데이터를 기반으로 종목의 성격 자동 부여.

### **2\. 세부 수행 작업 (Detailed Tasks)**

1. **상세 페이지 레이아웃:** 상단에 Glassmorphism 카드, 중앙에 차트, 하단에 타임라인 위젯 배치.  
2. **방사형 차트(Radar Chart) 구현:** fl\_chart 라이브러리를 사용하되, \*\*서로 단위가 다른 지표(PER은 배수, ROE는 %)를 0\~1 사이의 값으로 정규화(Normalization)\*\*하여 차트의 모양이 치우치지 않게 로직 구현.  
3. **스마트 태그 시스템:**  
   * PER \< 10 이면 \#저평가 태그 부여.  
   * ROE \> 15 이면 \#우량주 태그 부여.  
   * DividendYield \> 4% 이면 \#고배당 태그 부여.  
4. **거래 내역 타임라인:** ListView를 사용하여 매수/매도 이력을 시간순으로 리스트업하고 각 항목은 Glass 효과가 적용된 카드로 구현.

### **3\. 검증 및 디버깅 (Verification & Debugging)**

* **차트 정규화 검증:** PER이 5이고 ROE가 20%인 종목과 PER이 50이고 ROE가 5%인 종목을 비교했을 때, 차트가 한쪽으로 찌그러지지 않고 각 지표의 우열이 시각적으로 명확한지 확인.  
* **태그 중복/누락 체크:** 여러 조건에 부합하는 종목에 여러 개의 태그가 겹치지 않고 보기 좋게 나열되는지 확인.  
* **스크롤 성능:** 거래 내역이 많아질 때 상세 페이지 스크롤이 부드러운지 확인.

## **📄 phase5.md: Ads Optimization & Final Fail-safe**

### **1\. 개요 및 설계 원칙**

* **수익화:** AdMob 연동.  
* **User Experience:** 광고가 앱 사용을 방해하거나 멈추게 하지 않는 'Fail-safe' 로직 적용.

### **2\. 세부 수행 작업 (Detailed Tasks)**

1. **AdMob SDK 설정:** 안드로이드 AndroidManifest.xml과 iOS Info.plist에 App ID 등록 및 SDK 초기화.  
2. **배너 광고 위젯:** 화면 하단에 고정되는 BannerAdWidget 구현.  
3. **전면 광고(Interstitial Ad) 시스템:**  
   * 사용자가 '전략 저장' 버튼을 누를 때 광고를 로드하고 보여주는 함수 작성.  
   * **Fail-safe 로직(중요):** 광고 로드 함수에 timeout을 걸거나 onAdFailedToLoad 리스너를 사용하여, **광고 로드에 실패하면 유저를 기다리게 하지 않고 즉시 다음 화면(저장 완료 화면)으로 이동.**  
4. **빈도 제한(Frequency Cap):** SharedPreferences 등을 활용해 전면 광고 노출 후 1시간 내에는 다시 광고가 뜨지 않게 제어.  
5. **최종 배포 설정:** 앱 아이콘(Launcher Icon) 설정 및 스플래시 화면(Native Splash) 구성.

### **3\. 검증 및 디버깅 (Verification & Debugging)**

* **광고 실패 시뮬레이션:** **비행기 모드(네트워크 차단)** 상태에서 전략 저장 버튼을 눌렀을 때, 앱이 멈추지 않고 0.5초 내에 다음 단계로 넘어가는지 확인.  
* **테스트 ID 확인:** 개발용 빌드에서 반드시 구글 제공 **테스트 전용 광고 ID**가 사용되고 있는지 재확인.  
* **전체 유저 시나리오 테스트:** 앱 설치 \-\> Mock 데이터 로드 \-\> 필터 생성 \-\> 전략 저장(광고 확인) \-\> 알림 수신까지의 전 과정을 에러 없이 수행 가능한지 최종 보고.
